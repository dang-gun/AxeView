불확실 타입

감시자를 생성할때 자동으로 감시방식이 지정되도록 OverwatchingType.Unidentified을 사용할 수 있다.
이 타입을 쓰면 AxeView가 자동으로 판단하여 감시방식을 변경한다.

이 타입은 'AxeViewSupport' 라이브러리를 위해 만들어졌다.
직접 사용하기보다는 뷰를 기준으로 감시자가 없는 대상을 생성할 때 사용하는 것이 좋다.



1) 텍스트 노드(Text Node)

테스트 :
front ▶▶▶ {{TextNode_UnType}} ◀◀◀ back


*Html입력은 자동으로 h3으로 생성됨



2) 속성(attribute) - 값이 있는 속성
값이 있는 속성은 무조건 OverwatchingType.Monitoring으로 동작한다.

테스트 :
속성 테스트용 돔입니다.
개발자 툴을 열어 변경된 속성을 확인하세요~




3) 속성(attribute) - 이름만 있는 속성
이름만 있는 테스트 돔1입니다.



4) 속성(attribute) - Value 감시



속성에 'Unidentified'를 'OverwatchingOutputType'은 'String'로 변경된다.
지정된 'OverwatchingType'은 상황에 따라 변경될 수 있다.

속성에 'Unidentified'를 사용하면 다음과 같이 동작한다
- 속성이름이 'value'인 경우 : OverwatchingType.Monitoring_AttrValue
- 속성에 값이 없는 경우 : AxeViewMoveType.Attr_Valueless
- 위 경우를 제외하면 'OverwatchingType.Monitoring'로 동작한다.
